The Project

We, team Swagfish, are developing a web application game called TicTacToe however the main focus on this project is not the programming behind it, but the development behind it and to learn how to set up technical infrastructures that supports agile development.

The game

The game itself is a simple TicTacToe. It's a two player game, X and O, and the main focus is to place three marks in a row. The newest version of the game is always available. Just head out to TicTacToe and check it out.

Gradle

The project is also run with an open source build automation system called gradle. With gradle you can easily run scripts that allows you for example, compile your code create a runnable Jar file or even run unit tests. It can be a little bit hard to set up gradle and getting used to it, but once that part is over it makes things much easier.

Travis CI

The project also uses Travis CI, which is a continuous integration service used to build and test projects at GitHub. Travis automatically detects when a commit has been made and pushed to a GitHub repository that is using Travis. Each time that happens, it builds the project and runs all tests.

Heroku

The project also uses a cloud platform called Heroku. Heroku allows us to build, delived and monitor our project and to run in on the web so everyone can enjoy it.

Test Driven Development

The Test Driven Development is the process that the Swagfish dedicate their work to. When there's a new feature, it will be managed on a new feature branch and everything is protected. Whenever we finish a new feature, we push it to a developer branch, and from there, only one person has the right to push it to the master branch. (The code is loosely coupled and follows good object oriented design practices)
